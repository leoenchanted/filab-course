WebGL 图像处理

用 GPU 加速实现专业级胶片效果

为什么用 WebGL？

图像处理需要大量的像素运算。一张 4K 图片有 800 多万个像素，每个像素都要进行颜色矩阵乘法、曲线映射、模糊计算...如果用 JavaScript 在 CPU 上逐像素处理，可能要等好几秒。

WebGL 让浏览器可以直接调用 GPU 进行并行计算——800 万个像素同时处理，瞬间完成。

⚡
性能对比

CPU 处理：串行执行，一个像素一个像素计算，4K 图片可能需要 2-3 秒。
GPU 处理：并行执行，同时计算成千上万个像素，同样的 4K 图片只需要 16-33 毫秒（一帧的时间）。

纹理（Texture）

在 WebGL 中，图片被上传到 GPU 后称为"纹理"（Texture）。就像给 3D 模型贴图一样，我们可以对这些纹理进行各种采样和计算操作。

GLSL 着色器

GLSL 是在 GPU 上运行的程序语言。Film Lab 编写了多个"着色器"来处理不同的图像效果。每个着色器接收输入纹理，输出处理后的像素颜色。

CODE 
uniform sampler2D u_image;
in vec2 v_texCoord;
out vec4 outColor;

void main() {
  vec3 color = texture(u_image, v_texCoord).rgb;
  
  // 应用白平衡
  color *= u_wb;
  
  // 应用色彩矩阵
  color = clamp(u_matrix * color, 0.0, 1.0);
  
  outColor = vec4(color, 1.0);
}

渲染函数解析

当用户调整滑块时，render 函数会被调用。它把所有调整参数传递给 GPU，然后执行最终的合成着色器。

CODE 
render(width, height, params, isCompare) {
  this.canvas.width = width;
  this.canvas.height = height;
  
  this.drawTo(null, this.adjustProgram, () => {
    gl.uniform1f(uniforms.u_opacity, params.opacity / 100);
    gl.uniform1f(uniforms.u_exposure, params.exposure / 50);
    gl.uniform1f(uniforms.u_contrast, params.contrast / 50);
    // ... 更多参数
    gl.uniform1i(uniforms.u_is_compare, isCompare ? 1 : 0);
  }, {
    u_image0: this.texOriginal,
    u_image1: filmTarget.tex
  });
}

多通道渲染

复杂的胶片效果不能一次完成，需要分多个"通道"（Pass）渲染。每个通道输出到临时纹理，下一个通道读取上一个通道的结果，层层叠加。

第一步：基础处理

白平衡调整 → 色彩矩阵变换 → 曲线映射 → 黑白转换（可选）

输入：原始图片
输出：基础色调调整后的图片
第二步：光晕效果（Bloom）

提取高光区域 → 高斯模糊 → 叠加到原图

效果：让亮部产生柔和的"发光"感，模拟胶片的高光特性
第三步：颗粒效果

生成随机噪声纹理 → 高通滤波 → 按亮度混合

效果：添加胶片特有的颗粒质感，让数字照片更像胶片
第四步：最终合成

叠加光晕图层 → 叠加颗粒图层 → 应用用户调整参数

输出：最终显示在屏幕上的图像
