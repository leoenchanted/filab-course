状态管理的艺术

page.tsx 管理着应用的所有核心状态。让我们看看它用 React Hooks 管理了哪些数据：

CODE 
const [currentFilm, setCurrentFilm] = useState<string | null>(null);
const [params, setParams] = useState<AdjustParams>({...});
const [hasImage, setHasImage] = useState(false);
const [mode, setMode] = useState<Mode>('presets');
const canvasRef = useRef<HTMLCanvasElement>(null);
const glRef = useRef<WebGLProcessor | null>(null);
PLAIN ENGLISH

当前选中的胶片 ID（如 'kodak'），null 表示未选择

所有调整参数（曝光、对比度、色温等 12 个值）

布尔值：用户是否已上传图片

当前面板模式：'presets' 预设面板 或 'adjust' 调整面板

用 useRef 获取 Canvas DOM 元素的引用

用 useRef 持久保存 WebGL 处理器实例（不触发重渲染）

📝
useState vs useRef

useState 适合存储会改变并需要更新 UI 的数据（如当前选中的胶片）。useRef 适合存储不需要触发界面更新的数据（如 WebGL 实例），或者在多次渲染之间保持不变的引用。

FilmItem 组件详解

FilmItem 是一个纯展示组件，接收 film 数据、是否激活状态和点击回调，渲染一个胶片缩略图卡片。

CODE 
export default function FilmItem({ film, isActive, onClick }: FilmItemProps) {
  return (
    <div
      className={`film-item ${isActive ? 'active' : ''}`}
      onClick={onClick}
    >
      <Image src={`/films/${film.img}`} alt={film.name} />
      <span>{film.name}</span>
    </div>
  );
}
PLAIN ENGLISH

定义 FilmItem 组件，接收三个属性参数

返回 JSX 结构

外层 div，CSS 类名根据 isActive 动态变化

如果激活就加上 'active' 类，显示橙色边框

点击时调用父组件传入的 onClick 函数

使用 Next.js Image 组件加载胶片缩略图

显示胶片名称（如 "Portra 400"）
