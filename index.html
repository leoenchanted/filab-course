胶片模拟曾经是专业摄影师的专属工具，需要昂贵的 Lightroom 插件或 Photoshop 动作。Film Lab 把它带到了浏览器里——免费、即时、专业级。

WebGL 图像处理

用 GPU 加速实现专业级胶片效果

为什么用 WebGL？

图像处理需要大量的像素运算。一张 4K 图片有 800 多万个像素，每个像素都要进行颜色矩阵乘法、曲线映射、模糊计算...如果用 JavaScript 在 CPU 上逐像素处理，可能要等好几秒。

WebGL 让浏览器可以直接调用 GPU 进行并行计算——800 万个像素同时处理，瞬间完成。

⚡
性能对比

CPU 处理：串行执行，一个像素一个像素计算，4K 图片可能需要 2-3 秒。
GPU 处理：并行执行，同时计算成千上万个像素，同样的 4K 图片只需要 16-33 毫秒（一帧的时间）。

纹理（Texture）

在 WebGL 中，图片被上传到 GPU 后称为"纹理"（Texture）。就像给 3D 模型贴图一样，我们可以对这些纹理进行各种采样和计算操作。

GLSL 着色器

GLSL 是在 GPU 上运行的程序语言。Film Lab 编写了多个"着色器"来处理不同的图像效果。每个着色器接收输入纹理，输出处理后的像素颜色。

CODE 
uniform sampler2D u_image;
in vec2 v_texCoord;
out vec4 outColor;

void main() {
  vec3 color = texture(u_image, v_texCoord).rgb;
  
  // 应用白平衡
  color *= u_wb;
  
  // 应用色彩矩阵
  color = clamp(u_matrix * color, 0.0, 1.0);
  
  outColor = vec4(color, 1.0);
}

多通道渲染

复杂的胶片效果不能一次完成，需要分多个"通道"（Pass）渲染。每个通道输出到临时纹理，下一个通道读取上一个通道的结果，层层叠加。

第一步：基础处理

白平衡调整 → 色彩矩阵变换 → 曲线映射 → 黑白转换（可选）

输入：原始图片
输出：基础色调调整后的图片
第二步：光晕效果（Bloom）

提取高光区域 → 高斯模糊 → 叠加到原图

效果：让亮部产生柔和的"发光"感，模拟胶片的高光特性
第三步：颗粒效果

生成随机噪声纹理 → 高通滤波 → 按亮度混合

效果：添加胶片特有的颗粒质感，让数字照片更像胶片
第四步：最终合成

叠加光晕图层 → 叠加颗粒图层 → 应用用户调整参数

输出：最终显示在屏幕上的图像

胶片预设系统

如何用代码"模拟"真实胶片的色彩科学

胶片为什么看起来不一样？

每种胶片都有独特的"性格"。Kodak Portra 擅长肤色，Fuji 偏绿，CineStill 有标志性的红色光晕。这些差异来自胶片的化学配方和显影工艺。

数字模拟的核心是：用数学公式复制这些化学特性。Film Lab 使用了四种主要技术：色彩矩阵、曲线映射、光晕效果和颗粒生成。

🔢

色彩矩阵

3x3 矩阵重新映射 RGB 通道，改变整体色彩倾向

📈

曲线映射

非线性函数调整亮部和暗部的响应，控制对比度

✨

光晕效果

高斯模糊模拟胶片高光"渗开"的物理特性

🌾

颗粒生成

随机噪声模拟胶片银盐颗粒的质感

色彩矩阵变换

色彩矩阵 是最基础的色彩调整方式。通过 3x3 矩阵，我们可以重新混合红、绿、蓝三个通道的比例。

CODE 
// Kodak Portra 400 的色彩矩阵
matrix: [
  0.92, 0.05, 0.03,  // 新 R = 0.92*R + 0.05*G + 0.03*B
  0.00, 0.98, 0.02,  // 新 G = 0.00*R + 0.98*G + 0.02*B
  0.04, 0.08, 0.88   // 新 B = 0.04*R + 0.08*G + 0.88*B
]
PLAIN ENGLISH

注释：这是柯达 Portra 400 胶片的色彩矩阵配置

矩阵开始

第一行：新的红色通道 = 92%原红 + 5%原绿 + 3%原蓝

第二行：新的绿色通道 = 0%原红 + 98%原绿 + 2%原蓝

第三行：新的蓝色通道 = 4%原红 + 8%原绿 + 88%原蓝

矩阵结束

💡
Portra 的"肤色保护"

观察上面的矩阵：绿色通道几乎保持不变（98%），红色通道略微降低（92%），蓝色通道明显降低（88%）。这种组合让肤色更暖、更自然，是 Portra 胶片备受人像摄影师喜爱的原因。

曲线映射

曲线映射 让胶片可以对不同亮度区域做不同的处理。比如让暗部更实、亮部更柔，这就是胶片特有的"宽容度"。

CODE 
// Fuji Pro 400H 的曲线
curve: [
  [0,   0],   // 输入 0 → 输出 0（纯黑保持）
  [40, 38],   // 输入 40 → 输出 38（暗部微压）
  [128, 135], // 输入 128 → 输出 135（中间调提亮）
  [220, 232], // 输入 220 → 输出 232（亮部提升）
  [255, 252]  // 输入 255 → 输出 252（纯白微压）
]
PLAIN ENGLISH

注释：Fuji Pro 400H 胶片的曲线配置

曲线控制点数组开始

第1点：输入0(纯黑) → 输出0，保持纯黑

第2点：输入40(暗部) → 输出38，略微压缩

第3点：输入128(中间调) → 输出135，提亮

第4点：输入220(亮部) → 输出232，明显提升

第5点：输入255(纯白) → 输出252，略微压缩

数组结束

🎨
Fuji 的" airy look"

Fuji 胶片的特点是亮部特别通透。看曲线：220 的输入被映射到 232 的输出，这意味着亮部被整体提亮，产生那种"空气感"。同时纯白微压（252）防止过曝死白。

胶片颗粒的秘密

胶片颗粒不是简单的"噪声"，而是有结构的。亮部颗粒细、暗部颗粒粗，彩色胶片还有色度颗粒和亮度颗粒的区别。

CODE 
// 频谱颗粒生成（Spectral Grain）
buildSpectralGrain(width, height, intensity) {
  // 1. 生成随机噪声纹理
  const noiseTex = this.ensureNoiseTexture(...);

  // 2. 高通滤波提取细颗粒
  const fineTex = this.buildHighpassLuma(noiseTex, ...);

  // 3. 可选：模糊产生粗颗粒
  if (softness > 0.001) {
    const blurred = this.blurTexture(noiseTex, softness);
  }

  // 4. 按亮度混合（亮部颗粒少，暗部颗粒多）
  return { mainTex, fineTex };
}
PLAIN ENGLISH

注释：频谱颗粒生成技术

方法定义开始

步骤1：生成基础随机噪声

步骤2：高通滤波提取高频细节（细颗粒）

步骤3：如果需要软颗粒效果，进行模糊处理

步骤4：根据图像亮度混合颗粒（亮部少暗部多）

🔬
为什么叫"频谱"颗粒？

真实胶片颗粒有不同的"尺寸"。高通滤波提取的是高频（细颗粒），模糊后的是低频（粗颗粒）。把不同频率的颗粒叠加，就能得到自然的颗粒结构——就像音频里的低音+高音组成完整音乐一样。

Film Lab 的后端架构

Film Lab 主要是一个前端应用，所有图像处理都在浏览器中完成。但它仍然需要一些后端功能：访客统计和管理员认证。这些功能通过 Next.js App Router 的 API Routes 实现。

🗄️
Supabase 数据库

托管的 PostgreSQL 数据库，存储访客记录

🎯
为什么选择 Supabase？

Supabase 是开源的 Firebase 替代品，提供 PostgreSQL 数据库、实时订阅、身份验证等功能。对于 Film Lab 这样的小项目，免费额度足够使用，而且不需要维护服务器。

访客是如何被记录的？

每当用户打开 Film Lab，前端会发送一个请求到 /api/visitors。后端会记录访客信息，同时返回当前总访问量。

每个访客会被分配一个唯一的 UUID，存储在 HTTP-only Cookie 中（有效期一年）。这样同一用户再次访问时，后端能识别出来，避免重复计数。这也是保护隐私的做法——不追踪个人，只记录访问。

// 从请求头中获取真实 IP
function getClientIP(request: NextRequest): string {
  // Cloudflare 代理
  const cfIP = request.headers.get('cf-connecting-ip');
  if (cfIP) return cfIP;

  // 标准代理头
  const forwardedFor = request.headers.get('x-forwarded-for');
  if (forwardedFor) return forwardedFor.split(',')[0].trim();

  // Nginx 代理
  const realIP = request.headers.get('x-real-ip');
  if (realIP) return realIP;

  // 本地开发
  return '127.0.0.1';
}

简单的管理员认证

Film Lab 有一个简单的管理后台（/admin），用于查看访客统计数据。进入需要用户名和密码验证。

CODE 
// 登录验证
export async function POST(request: NextRequest) {
  const { username, password } = await request.json();

  // 从环境变量读取凭据（默认 zzh/060312）
  const adminUser = process.env.ADMIN_USERNAME ?? 'zzh';
  const adminPass = process.env.ADMIN_PASSWORD ?? '060312';

  if (username === adminUser && password === adminPass) {
    // 生成简单 token：用户名:时间戳 的 base64
    const token = Buffer.from(`${username}:${Date.now()}`)
      .toString('base64');

    const response = NextResponse.json({ success: true });
    response.cookies.set('admin_token', token, {
      httpOnly: true,
      secure: process.env.NODE_ENV === 'production',
      maxAge: 60 * 60 * 24 // 24小时
    });
    return response;
  }

  return NextResponse.json({ error: 'ACCESS DENIED' }, { status: 401 });
}

环境变量的作用

敏感信息（数据库密钥、管理员密码）不应该写在代码里，而是通过环境变量传入。Film Lab 需要这些环境变量：

SUPABASE_URL Supabase 项目地址，如 https://xxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY 服务端密钥，有完整数据库访问权限（不要泄露！）
ADMIN_USERNAME 管理员用户名（可选，默认 zzh）
ADMIN_PASSWORD 管理员密码（可选，默认 060312）
🛡️
Service Role Key 的危险性

Supabase 的 Service Role Key 拥有绕过所有 Row Level Security 的权限。这意味着任何人拿到这个密钥就能读写你的整个数据库。永远只把它放在服务器端，永远不要暴露在浏览器代码中！
